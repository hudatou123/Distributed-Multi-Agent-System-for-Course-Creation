# Internal Course Style Guide (v1)

> This is an internal organizational standard. It is NOT published anywhere on
> the public web, so if a generated course follows it, the content must have
> come from this knowledge base via MCP — a useful way to verify the MCP
> integration is actually working.

## The EDGE Format (mandatory)

Every course module MUST be structured in four parts, in this exact order:

1. **E — Explain:** State the concept in plain language and why it matters.
2. **D — Demonstrate:** Give one concrete worked example.
3. **G — Guide:** Provide a short step-by-step the learner can follow themselves.
4. **E — Evaluate:** End with 2–3 check-your-understanding questions.

## Tone and length rules

- Write for a motivated beginner. Avoid unexplained jargon.
- Each module's **Explain** section must be **at most 120 words**.
- Always include a one-sentence "Key takeaway" in **bold** at the end of every module.

## Forbidden

- Do not start a module with a dictionary-style definition.
- Do not use the phrase "in today's fast-paced world".
